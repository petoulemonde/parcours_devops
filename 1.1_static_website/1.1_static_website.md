# 1.1. Static website

Dans ce projet, je souhaite apprendre à conteneuriser un site web, pour pouvoir demain peut-être mieux maitriser le déploiement de mon site web.

Objectifs 🎯 : 
- apprendre un utiliser nginx (célèbre logiciel)
- créer un *bind mount* pour modifier mes fichiers localement et qu'ils soient automatiquement prises en compte dans mon conteneur.

Sources utilisées : 
- [Beginner’s Guide - Nginx](https://nginx.org/en/docs/beginners_guide.html)

Homelab utilisé : [v1](https://github.com/petoulemonde/homelab/releases/tag/v1)

## Commandes

### Préparer le dossier de travail
1. Créer mon volume de travail : 
```bash
mkdir app/
```

2. Créer mon index de site web : `app/index.html` : 
```html
<!DOCTYPE html>
<html lang="fr">

    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Document</title>
    </head>

    <body>

        <p>Hello world 🐕!</p>
        
    </body>

</html>
```

3. Tester un *bind mount* : 
```bash
podman run -it -v $(pwd)/app:/app alpine sh # Lance un container alpine intéractif
cd app # ouvre le dossier app
ls # si tout a bien marché, *index.html* devrait apparaitre
vi index.html # afficher le fichier avec VIM

exit # Fermer alpine
```

4. Créer le dockerfile

## Commandes
1. Construire l'image
```bash
podman build . -t website:v1
```

2. Lancer l'image : 
```bash
podman run -d -p 8080:80 -v $(pwd)/app:/usr/share/nginx/html --name web website:v1
# fonctionne aussi avec : ~/app plutôt que $(pwd)>/app
```
- `--name web` : désigne le nom du conteneur

3. Se connecter au site : ouvrez notre navigateur et tapez : `http://localhost:8080`.
Vous devriez être accueilli par `Hello world 🐕!` 🎊

4. Modifiez votre fichier `index.html` depuis votre ordinateur en dehors du WSL, sauvegardez puis rafraichissez votre navigateur, vous verrez la page avec les modif' approtées. C'est pas grand chose, mais ça donne un petit effet Waouh 🙂
