# Mon premier dockerfile

Mon objectif est de transformer le script de mon expérience 0.5_premeir_container en un dockerfile. 
Cette expérience de permettra d'apprendre à créer un dockerfile?

Source : [TP1 - Image Docker basique avec Debian 12 - Stéphane Robert](https://github.com/stephrobert/containers-training/blob/main/00-Docker/01-debian12-basique/01-contruction-1ere-image.md) 

Homelab utilisé : [v1](https://github.com/petoulemonde/homelab/releases/tag/v1)

## Commandes

1. Naviguer dans ce dossier

2. Construire l'image : `podman build . -t premier_dockerfile`
Vérifier que l'image a bien été créée : `podman images` (l'image nommée *premier_dockerfile* devrait apparaitre)

NB : 
- `-y` : sans ça, l'image attend une validation utilisateur. EN mode détachée, cette input utilisateur attendu crée une sortie du container
- `--break-system-packages` : permet au container d'aller chercher des packages en externe lors de son build. Sans ça, le container remonte une erreur.

3. Run l'image : `podman run -it premier_dockerfile`
NB : `-it` : démarrage en mode intéractif, sinon la vache ne dira rien car sortie pas attachée

Ca marche 🎊 Ma vache me dit bonjour ! 
En l'absence d'autres actions à réaliser, le ctonainer se stoppe d'emblée.

Pour suppriemr le conteneur arreté : 
- `podman ps -a` : identifier le nom du conteneur (*PORTS* = *localhost/premier_container*) -> regarder la colonne *NAMES* 
- `podman rm <nom>` : supprime le container arreté
